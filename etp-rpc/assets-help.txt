createasset
createasset

Parameters (optional)
-d or [--description] The asset description.
-i or [--issuer] The did symbol specified its issuer.
-n or [--decimalnumber] The asset amount decimal number.
-r or [--rate] The percent threshold value when you secondary issue.
-s or [--symbol] The asset symbol/name. Global unique.
-v or [--volume] The asset maximum supply volume.
Parameters (positional)
ACCOUNTNAME Account name.
ACCOUNTAUTH Account password/authorization.
params:[
    "ACCOUNTNAME",
    "ACCOUNTAUTH"
]
Returns
Object - the asset just created

Example

// Request
curl -X POST -d '{
    "id":25,
    "jsonrpc":"2.0",
    "method":"createasset",
    "params":[
        "test",
        "123456",
        {
            "description": "test asset",
            "issuer": "testdid01",
            "decimalnumber": 8,
            "symbol": "MVS.TST",
            "volume": 10000000,
            "rate": -1
        }
    ]
}' http://127.0.0.1:8820/rpc/v3

// Response
{
    "id" : 25,

    "result" :
    {
        "address" : "",

        "issuer" : "testdid01",
        "maximum_supply" : 10000000,

validatesymbol
Parameters (optional)
-c or [--cert] If specified name of cert type, then validate cert symbol. Default is not specified.
Parameters (positional)
ACCOUNTNAME Account name.
ACCOUNTAUTH Account password/authorization.
SYMBOL Symbol to be validated.
params:[
    "ACCOUNTNAME",
    "ACCOUNTAUTH",
    "SYMBOL"
]
Returns
status: available, exsited, no_permission or forbidden.

Example

// Request
curl -X POST --data '{
    "id":7,
    "jsonrpc":"2.0",
    "method":"validatesymbol",
    "params":[
        "test",
        "123456",
        "ERC20.EE"
    ]
}' http://127.0.0.1:8820/rpc/v3
